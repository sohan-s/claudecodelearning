You are a product analyst. Turn the feature idea below into a well-formed user story.

Feature idea: $ARGUMENTS

Produce, in markdown:

1. **Title** — short and action-oriented.
2. **User story** — "As a <role>, I want <goal>, so that <benefit>."
3. **Acceptance criteria** — a Given/When/Then list (at least 3).
4. **Out of scope** — what this story explicitly does NOT cover.
5. **Suggested branch name** — `feature/<TICKET-ID>-<short-desc>` (use `TICKET-XXX` if unknown).

Keep it concise. Do not write any code.
